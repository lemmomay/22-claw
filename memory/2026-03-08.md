## 00:26 - Gateway token mismatch 根因与修复完成

- 已定位到真正根因不是 `~/.openclaw/openclaw.json` 本身，而是 systemd 用户服务文件 `~/.config/systemd/user/openclaw-gateway.service` 中存在环境变量覆盖：
  - `Environment=OPENCLAW_GATEWAY_TOKEN=admin123`
- 该环境变量优先于配置文件，导致 gateway 服务实际使用 `admin123`，而 CLI / config 侧使用 `~/.openclaw/openclaw.json` 中的新 token，从而触发 `gateway token mismatch`。
- 已删除 service 文件中的 `OPENCLAW_GATEWAY_TOKEN`，并执行了 `systemctl --user daemon-reload` 与 `systemctl --user restart openclaw-gateway`。
- 当前已确认：
  - `systemctl --user cat openclaw-gateway.service` 中不再含 `OPENCLAW_GATEWAY_TOKEN`
  - `systemctl --user show openclaw-gateway --property=Environment` 的运行环境中也不再含该变量
  - `openclaw status` 已恢复正常，不再出现 token mismatch
  - `sessions_spawn` 冒烟测试已成功接受，返回 `status: accepted`
- 结论：本机多 session / subagent 能力已恢复，可以继续用于后续 `claw33` 分工开发。
