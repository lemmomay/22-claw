## 18:19 - Sing-box 架构设计迭代 #4

**本次分析：** 配置验证与回滚模块

**核心发现：**
- fscarmen 的升级流程：备份 → 替换 → 验证 → 回滚（如果失败）
- 使用 `.bak` 文件作为临时备份，成功后删除
- 双重验证：升级后验证，回滚后再次验证
- 状态检测使用 `systemctl is-active` 而非 `ps` 命令

**设计原则：**
1. 原子操作 - 每步都可逆
2. 失败快速恢复 - 自动回滚
3. 备份即保险 - 重大操作前自动备份
4. 验证驱动 - 主动检测服务状态

**接口设计：**
- `validate_config()` - 配置验证（sing-box check + 端口冲突 + 证书检查）
- `backup_config()` - 带时间戳的配置备份
- `restore_config()` - 配置恢复
- `upgrade_with_rollback()` - 带自动回滚的升级流程
