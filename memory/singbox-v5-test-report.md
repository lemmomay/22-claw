## 🐛 发现的问题

### 1. sing-box -C 参数问题 ⚠️

**问题描述：**
- sing-box 的 `-C` 参数（加载配置目录）合并逻辑不符合预期
- 多个文件中的 `inbounds` 数组无法正确合并
- 导致 "EOF" 或 "duplicate tag" 错误

**临时解决方案：**
- 使用单文件配置模式（`-c config.json`）
- 配置生成器生成完整的单文件配置

**长期解决方案：**
- 实现自定义配置合并脚本
- 或研究 sing-box 的正确配置分片方式

### 2. yq 数组操作问题 ✅ 已解决

**问题描述：**
- 初始版本的 `node_add` 使用 `yq eval-all` 导致 YAML 结构错误

**解决方案：**
- 改用 `yq eval -i ".nodes += [...]"` 语法
- 先将节点数据转为 JSON，再追加到数组

### 3. 旧配置文件干扰 ✅ 已解决

**问题描述：**
- `config/` 目录下的旧文件（trojan.json, merged.json）被 sing-box 加载
- 导致 "duplicate outbound tag" 错误

**解决方案：**
- 清理所有旧配置文件
- 只保留 `base/` 和 `inbounds/` 子目录中的文件
